A brief explanation of APIs...

An API can be described as a middleman.
If you request information in the correct way, he will send the information requested back to your computer, and you will be all the wiser. You might ask, why does he not show you how to get the information yourself? Each connection he has his hands on is complicated and would take you much more time to learn thank just to ask the middleman. Plus, the connection might not want to open up its secret crafts.



You might be wondering what covert means are required to contact such a powerful middleman...

Luckily, this is not as complicated as it can be in real life. All you need need to do is learn about the site's API. You can even use this tutorial about the GoodReads API to gain a better grasp of APIs in general and how to interact with them.

Should the enigma of APIs spiked your curiosity, you can either click here or continue reading on.

API Keys

An API key is a secret token which is submitted alongside a request to an API in order to identify where the request came from.

Sometimes the organizations that owns the API wants to identify the source of a request to their API in order to control access. For example, they can restrict access to certain API actions based on who's performing the request. For companies which make money from selling such services, it's also a way of tracking who's using their API and how often for billing purposes. Furthermore, by blocking a key, they can prevent users from making too many web service calls to their API and therefore overwhelming their server.

The GoodReads API Key...

In order to get your very own GoodReads API key, you will first need to create an account on GoodReads.
Then, you will need to go to the GoodReads API and click on register your app. After this, you will need to provide the Application name, Company name, Application URL(the url that will be accessing the Goodreads API), Callback URL(the url that contains the callback to a GoodReads API function), and Support URL(the URL that contain functions that help either the application URL or the callback URL complete their tasks). Then you need to read through the terms of service. Finally, you can click Create Key and you will be directed to a page containing your API Key.

GoodReads API Methods

Here we will quickly take a look at three GoodReads API methods,
which we will place onto our own website in the upcoming section .

reviews.list()

This function lists the books a user has on their shelf. It calls the Goodreads server and then requests the user's shelf information. It then puts the designated user's shelf information and returns it to the calling site. Its parameters require an API key, the user's id whose shelf is requested, and the number of shelves that should be returned. In this tutorial you will recieve an in-depth look at how all of this takes place.

review.show()

This function finds a review on the GoodReads website. You can then link the review to your website and allow others to see the content. The benefit of using this function is that if the GoodReads user realizes they have made a grammar mistake or could be more articulate in phrasing their thoughts, the newest version will appear on your website whenever the function is called. It takes as parameters the review's id and an API Key.

Find.Group()

This function searches both group names and their descriptions for a word or phrase the user wants to find. This function will take a string, which will hold the words the user wants to search, alongside the number of pages of results that should be returned and our API Key.

An Overview of Get and Post Requests

On GoodReads, we have the opportunity to create both post and get requests. A Get request is one where we ask the GoodReads server for information. In order to get exactly the information we are searching for, we need to send certain parameters as defined for that particular Get method. The Post request is very similar to the Get request, except that we put information on the GoodReads server. In GoodReads we need to use OAUTH, which is a third-party tool that allows users to log in. OAUTH is used so that people are associated with a user account when they post and therefore are less likely to post harassment or can be stopped more easily if they do. It is fully documented by GoodReads, so please read up upon it here.

Setting Up the HTML and CSS for Input and Output

Setting up the HTML and CSS for input and output can be quite simple if you already know and understand HTML and CSS. However, in order to be certain the rest of this tutorial is very clear, I will quickly go over the input and output elements.
The element of interest is the User's ID. This section is nested in an input element. The id is what we will use to declare which input element we want to call data from. Also, note that for testing it is often important to not have to refill in different text boxes, because that becomes tedious and can take up a lot of time if you need to rerun the tests multiple times before they succeed. Therefore, I have provided a preordained value to the input field.
The next element of interest is the div that contains the id and class where we want to output information. These fields do not require an output wrapper, as might be expected, but can simply work as divs. By delaring the div's id, the return function will be able to call up that div and output the information in its declared textfield.

Setting Up The Communication With The GoodReads Server

Javascript and JSON will be taking on the role of Postman and Postal Service in the process of doing the magic behind the submit. Javascript will be taking the information that the user provides, as well as some internal information, and then creating a JSON object. Javascript will then send this JSON object to the URL and requesting a response. After this, it will parse the response and send it back to the website. We will be going through each of these steps and seeing exactly how this is done.

Setting up the Variables

The variables that we need to create are the same as the parameters of the GoodReads function we are trying to call. The GoodReads function we are looking at initially is the Review.list() function, which requires the developer key (GoodReads synonym for API key) and the GoodReads ID for the user. There is other information that can be added, such as the ordering of the results or the results per page, but these are optional and can easily be added in the future in the same way we add the API key and GoodReads ID. Once we have our variables together, we concatenate them in the order that they are requested by the site, which in this case was the url provided for the function, the user id, and the API key.

The Request...

In order to make the request, we need to send open to our Request variable and provide three pieces of information. The first is if we want to get or post information. In this example, we want to get information, so we write "GET". Then it requires to concatenated url that we created in our variables section. The final parameter is a boolean that says whether this is a syncronous or an asynchronous request. A synchronous request blocks everything until the request returns. An asynchronous request does not do this, so it allows other javascript actions to take place, but it does require an event handler to notice the response came in. Then we need to add a listener for when the request returns and says its information is loaded. We will go over this in the next section. After this, the request can be sent, through calling the Request.send() function. Finally, we add the function preventDefault() because if you do not, the browser can refresh and lose all of the information.

The Response...

In order to read the response we need to create a listener that awaits a response from the GoodReads server. This response will provide us with a status that declares whether the response went well (numbers between 200 and less than 400), or it went awry (any other number). If we find the status to be between 200 and 400, we can parse the response and put it into a variable. Since GoodReads sends back XML data, we need to use the XML to JSON function that we mentioned above to make the response into a JSON object. This JSON object is then stored in the Response variable where we will send it's information onwards in the next section.

Making the Response Visible for the Site Viewer

In the last section we received a json object and put it in the variable Response. In order to make this object provide the viewer with the information they requested, we need to call its parts and put them into our HTML elements. To do this, we look at the response and see where information is that we are searching for. The information was returned in a series of arrays, so we found the objects that we wanted in those arrays and called their properties.
